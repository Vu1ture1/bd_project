1. Таблица "Роли" ("ROLES") - содержит в себе информацию о роли пользователя
   Связана с таблицей "Пользователи" связью "Один ко многим" (одну роль может иметь множество пользователей, а один пользователь может иметь только одну роль)
   Содержит в себе следующие поля:
   - name: название должности - тип поля строка(до 30 символов)
   - salary: зарплата роли - тип поля строка(до 30 символов)

2. Таблица "Пользователи" ("USER") - содержит в себе информацию о пользователях
   Связана с таблицей "Информация о пользователе" связью "Один к одному" (у каждого пользователя есть личная информация и у каждой личной информации есть только один пользователь) 
   Связана с таблицей "Заказы" связью "Один ко многим" (в одном заказе может участвовать один пользователь, но у одного пользователя может быть несколько заказов)
   Связана с таблицей "Корзина" связью "Один к одному" (у каждого пользователя есть корзина и у каждой корзины есть только один пользователь)
   Связана с таблицей "Журнал" связью "Один ко многим" (одному пользователю могут принадлежать множество записей журнала, а одной записи может соответствовать только один пользователь)
   Связана с таблицей "Типы действий" связью "Многие ко многим" через промежуточную таблицу "Журнал" (одному пользователю могут соответствовать множество  типов записей журнала, а одному типу записи может соответствовать много пользователей)
   Связана с таблицей "Отзыв" связью "Один ко многим" (так как одно отзыв относится к одному пользователю, а у пользователя может быть несколько отзывов)
   Связана с таблицей "Предмет" связью "Один ко многим" (так как один предмет относится к одному пользователю, а у пользователя может быть мн-во выставленных предметов)

   Содержит в себе следующие поля:
   - username: псевдоним пользователя - тип поля строка(до 30 символов)
   - password: пароль пользователя - тип поля строка(до 30 символов)
   - post: должность пользователя (Внешний ключ)
   - user_info: личная информация пользователя(Внешний ключ)

2. Таблица "Личная инфа пользователя" ("USER_INFO") - содержит в себе личную информацию о пользователе
   Содержит в себе следующие поля:
   - name: псевдоним пользователя - тип поля строка(до 30 символов)
   - surname: пароль пользователя - тип поля строка(до 30 символов)
   - email: мыло - тип поля строка(до 30 символов)
   - phone_number: номер телефона - тип поля строка(до 30 символов)
   - adress: адресс - тип поля строка(до 200 символов)

4. Таблица "Журнал" ("LOG") - содержит в себе информацию о журнале действий
   Связана с таблицей "Пользователь" связью "Один ко многим".
   Связана с таблицей "Тип действия" связью "Один ко многим" (одному типу действия может соответствовать множество записей журнала, а одной записи может соответствовать только один тип действия)
   Содержит в себе поля:
   - log_type: поле типа действия. (Внешний ключ)
   - user: поле пользователя. (Внешний ключ)
   - date: дата и время действия - тип поля date

5. Таблица "Типы действий" ("LOG_TYPE") - содержит в себе типы действий.
   Связана с таблицей "Журнал" связью "Один ко многим".
   Содержит в себе поля:
   - description: название действия - тип поля строка (до 20 символов).

6. Таблица "Корзина" ("CART") - содержит в себе корзину пользователя.
   Связана с таблицей "Пользователь" связью "Один к одному".
   Связана с таблицей "Предмет" связью "Многие ко многим" через промежуточную таблицу "Предмет в корзине" (одной корзине могут соответствовать множество предметов, а один предмет может содержаться в мн-ве корзин)
   Содержит в себе поля:
   - user: ссылка на владельца корзины(внешний ключ).

7. Таблица "Предмет" ("ITEM") - содержит в себе предмет выставленный пользователем.
   Связана с таблицей "Пользователь" связью "Один ко многим" (у каждого пользователя есть мн-во предметов но у каждого предмета есть только один пользователь) 
   Связана с таблицей "Корзина" связью "Многие ко многим" через промежуточную таблицу "Предмет в корзине" (одной корзине могут соответствовать множество предметов, а один предмет может содержаться в мн-ве корзин)
   Связана с таблицей "Заказы" связью "Многие ко многим" через промежуточную таблицу "Предмет в заказе" (одному заказу могут соответствовать множество предметов, а один предмет может содержаться в мн-ве заказов)
   Связана с таблицей "Категории" связью "Многие ко многим" через промежуточную таблицу "Категория предмета" (одному предмету могут соответствовать множество категорий, а одна категория может содержаться в мн-ве предметов)
   Связана с таблицей "Запрос" связью "Один к одному" (одному предмету может соответствовать только один запрос, а одному запросу может соответствовать один предмет)
   Связана с таблицей "Отзыв" связью "Один ко многим" (одному предмету может соответствовать мн-во отзывов, а одному отзыву может соответствовать один предмет)
   - item_name: название предмета - тип поля строка(до 30 символов)
   - seller: поле продавца. (Внешний ключ)
   - description: описание - тип поля строка
   - price: цена - тип поля float
   - listing_date: дата добавления - тип поля date
   - status: статус одобрения - тип поля bool
   - items_count: кол-во вещей для покупки - тип поля int


8. Таблица "Запрос" ("REQUEST") - содержит в себе запрос для модерации.
   Связана с таблицей "Предмет" связью "Один к одному".
   Содержит в себе поля:
   - item: ссылка на модерируемый предмет(внешний ключ) 
   - decision: ответ модерации(внешний ключ).

9. Таблица "Заказ" ("ORDER") - содержит в себе заказ пользователя.
   Связана с таблицей "Пользователь" связью "Многие к одному" (у одного пользователя может быть мн-во заказов а у заказа только один пользователь)
   Связана с таблицей "Предмет" связью "Многие ко многим" через промежуточную таблицу "Предмет в заказе" (одному заказу могут соответствовать множество предметов, а один предмет может содержаться в мн-ве заказов)
   Связана с таблицей "Купоны" связью "Многие ко одному" (одному заказу могут соответствовать один купон, а один купон может содержаться в мн-ве заказов)

   Содержит в себе поля:
   - buyer: ссылка на того кто оформил заказ(внешний ключ).
   - total: общая сумма заказа с исп купоном - тип поля float.
   - creation_date: дата создания заказа - тип поля date.
   - Coupon: ссылка на используемый купон(внешний ключ).

10. Таблица "Купон" ("COUPONS") - содержит в себе купоны на любые заказы.
   Связана с таблицей "Заказ" связью "Один к многим" (у одного заказа может быть один купон а у купона мн-во заказов)
   
   Содержит в себе поля:
   - buyer: ссылка на того кто оформил заказ(внешний ключ).
   - total: общая сумма заказа с исп купоном - тип поля float.
   - creation_date: дата создания заказа - тип поля date.
   - Coupon: ссылка на используемый купон(внешний ключ).

11.  Таблица "Категория" ("CATEGORY") - содержит в себе категорию.
   Связана с таблицей "Предмет" связью "Многие к многим"  через промежуточную таблицу "Категория предмета" (одному предмету могут соответствовать множество категорий, а одна категория может содержаться в мн-ве предметов)
   
   Содержит в себе поля:
   - category_name: название категории - тип поля строка
